ABOS Offer Configuration

Designed, created, tested, and launched an internal full-stack infrastructure for Amazon to streamline the configuration of ABOS (Audible Buying Options Service) offers, which allowed Audible employees to save time creating, adding, updating, and displaying offers.

  • javascript
  • typescript
  • html
  • css
  • react
  • git
Live Demo

The plan is from the Web Browser, the user hits the DNS/URL. The used Route 53 to create the DNS. Go through a Midway Authentication to authenticate the user Once the user is authenticated. The user can access the website hosted on Cloudfront. Cloudfront improves performance. It allows us to have https on the website, which makes the website more secure. It will point to our S3 bucket which contains our website files. The HTML and static content is deployed and stored in S3.

I designed the UI of the app, to create a foundation before using it.

The app makes it very straightforward for the user, in this case an Audible, or Amazon employee to create an offer by providing the customer's id, the book's asin number, the marketplace used, and the current device used by the customer, as requirement. And optionally the user will also provide the offer type, and program type.

Once the offer is created, we get the list of offers of that specific customer. Each offer is displayed with the offer's buyability, id, price, and offer type.

The app was made using Java for the backend, and react, javascript, html and css, for the first version of the UI. It is also responsive to different screen sizes.

The user is notified of the required input, if he tries to submit the form without the proper information.

The app was made using Java for the backend, and katal, react, typescript, html and css, for the second version of the UI.

This second version of the app is also responsive to different screen sizes.
